# 06 高级篇整理

## 这一章在讲什么

课程 06 的重点不再只是“换一种 fusion 模块”，而是开始进入更接近研究和实际项目的几个高级主题：

- 用 ViT 当 backbone
- 怎么利用预训练模型
- Lightning 里怎么做更合理的优化策略
- 推理阶段怎么靠 TTA 再抬一点指标

如果 05 更像“融合模块升级”，  
那 06 更像：

> 从模块设计走向训练策略、预训练 backbone 和实验技巧。

---

## 1. 从 CNN 到 ViT，这一章想让你建立什么感觉

课程里先做了一件很重要的认知切换：

- 前面几章你更熟的是 CNN 编码器
- 这一章开始告诉你：编码器不一定非要是卷积网络

传统 CNN 的特点是：

- 通过局部卷积逐步扩大感受野
- 更偏局部到全局

ViT 的特点是：

- 先把图像切成 patch
- 把 patch 看成 token
- 再通过自注意力直接建模全局关系

所以课程 06 想建立的第一个核心意识是：

> backbone 不只是“提特征的黑盒子”，它的建模方式会直接影响多模态任务的上限。

---

## 2. 为什么 ViT 会被拿来做 RGB-D 编码器

课程里强调了 ViT 在多模态任务上的几个优势：

### 2.1 全局建模能力更强

CNN 更像“邻域一点点往外看”。  
ViT 通过自注意力，天然更擅长：

- 远距离区域关系
- 全局上下文
- 场景整体布局

这对室内场景分割尤其重要，因为：

- 桌子和椅子常常在空间上一起出现
- 墙、地面、天花板的关系有明显全局结构

### 2.2 预训练迁移能力强

课程里虽然是教学版写法，但核心思想很真实：

- backbone 如果已经在大规模数据上预训练过
- 下游任务通常会更稳、更强

这就是为什么课程后面开始讲：

- Freeze
- Fine-tune
- 分层学习率

### 2.3 Patch 级表示更适合多模态对齐

这一点是多模态里特别关键的。

如果 RGB 被表示成 patch token：

- Depth 也可以转成 patch token
- 然后在 token 级别做交互

这就为后面的 MFM（Multi-modal Fusion Module）埋下了路。

## 4. Freeze、Fine-tune 和分层学习率到底在解决什么

### 4.1 Freeze

冻结的意思是：

- backbone 参数不更新
- 只训练融合模块、解码器、分类头

优点：

- 更稳
- 训练更省资源
- 数据少时不容易把预训练知识破坏掉

缺点：

- 适应下游任务的能力有限

### 4.2 Fine-tune

微调的意思是：

- backbone 也参与训练
- 但通常学习率要更小

优点：

- 更容易适配当前任务
- 上限通常更高

缺点：

- 更吃资源
- 训练更敏感

### 4.3 分层学习率（Layer-wise Learning Rate Decay）

课程里这一段非常重要。

它背后的直觉是：

- 靠近输入的层学到的是更通用的视觉知识
- 靠近输出的层更接近当前任务

所以常见做法是：

- 底层学习率更小
- 高层学习率更大
- 任务头（head）学习率最大

课程里的简化版本是按两组分：

```python
param_groups = [
    {"params": head_params, "lr": self.hparams.lr},
    {"params": backbone_params, "lr": self.hparams.lr * 0.1},
]
```

你可以把这看成：

- head：主要负责“学这门新任务”
- backbone：主要负责“轻微适配，不要学坏”

## 5. Warmup + Cosine 到底在帮什么忙

这一章不是只想让你会抄调度器代码，而是让你理解训练动态。

### 5.1 Warmup

训练刚开始时，如果一上来就用比较大的学习率，容易：

- loss 抖动
- 梯度不稳定
- 训练直接起飞后又崩掉

所以 Warmup 做的是：

- 先从小学习率起步
- 再逐步升到目标学习率

本质上是在说：

> 训练开始时先别太猛，先把模型带起来。

### 5.2 Cosine Annealing

后期再用余弦衰减，让学习率慢慢降下来。

这在直觉上就是：

- 前期探索
- 中期稳定学习
- 后期细收敛

所以 `Warmup + Cosine` 的组合通常会比固定学习率更稳。

如果你以后在 TensorBoard 里看学习率曲线，这一章就是最好的理论背景。

## 6. TTA 为什么常常能白捡一点分

TTA（Test-Time Augmentation）是课程 06 很实用的一块。

它的意思不是训练时增强，而是**推理时也做多种输入变换**，然后把预测融合起来。

常见做法：

- 多尺度
- 水平翻转
- 概率平均

课程里给的思路是：

1. 把同一张图用不同尺度推理
2. 再做翻转版本推理
3. 把各次输出的概率图平均
4. 最后再 `argmax`

它为什么常常有效：

- 不同尺度看到的物体大小不同
- 翻转能减轻方向偏置
- 概率平均比直接投票更平滑

课程里说常见提升有 `1-3% mIoU`，这个量级是合理的，但代价是：

- 推理更慢
- 部署时不一定适合

所以 TTA 更像：

- 论文评测技巧
- 离线验证技巧

而不是默认部署方案。

## 7. MFM 是什么，为什么课程只点到为止

课程 06 里提到的 MFM（Multi-modal Fusion Module），本质上是在更高级的特征空间里做多模态融合。

这里的关键变化是：

- 不再只是卷积特征图级别融合
- 而是进入 patch token / Transformer 特征空间里的交互

可以把它理解成：

- 05 章还主要是卷积风格的注意力融合
- 06 章开始把你往 Transformer 风格的跨模态交互上带

但课程本身也很诚实：

- 教学代码只是简化版
- 真正研究级的 MFM 会复杂很多

所以这一节你现阶段最重要的不是“立刻会写”，而是先记住方向：

> 未来更强的多模态融合，往往不是简单拼接，而是在更强 backbone 的特征空间里做交互。
