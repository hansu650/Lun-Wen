# 飞天闪客笔记补充：第 3 节结尾没展开的内容

这一页其实是在讲一件事：

- 神经网络训练时常见的问题有哪些
- 常见的解决办法分别是什么

也就是说，这些词不是彼此独立的，而是围绕“怎么把网络训练稳、训练快、训练好”展开的。

## 一、先看左边：训练里常见的 4 个问题

### 1. 梯度消失

- 反向传播时，梯度要一层一层往前传
- 如果每一层传回去的梯度都小一点，连乘很多次以后就会越来越小
- 小到前面那些层几乎收不到有效更新
- 结果就是：靠近输入的层学不动，训练很慢，深层网络尤其明显

可以把它想成：

- 本来每层传回来的是 `0.5`
- 连乘很多层以后就变成了非常接近 `0`
- 前面的参数几乎不更新

### 2. 梯度爆炸

- 和梯度消失相反
- 如果反向传播时梯度不断被放大，连乘很多次以后就会变得非常大
- 结果是参数更新步子太大
- loss 可能突然变得很大，训练不稳定，甚至直接变成 `NaN`

可以把它想成：

- 本来每层传回来的是 `2`
- 连乘很多层以后就会变成特别夸张的数
- 一次更新就把参数“甩飞了”

### 3. 收敛速度

- 收敛速度就是：loss 下降得快不快，模型多久能学到比较好的结果
- 不是说只要能降就行，而是要尽量更快、更稳地降

收敛慢通常表现为：

- 训练很多轮 loss 还是降得很慢
- 指标提升很慢
- 一直在局部区域来回震荡

### 4. 计算开销

- 就是训练一个模型到底要花多少算力、显存、时间
- 包括一次前向传播和反向传播的成本
- 也包括整个训练周期需要多久

在实际做实验时，这个问题很重要，因为：

- 模型再好，跑不动也没法做
- 显存不够时 batch size 会受限
- 训练时间太长会影响做消融实验和调参

## 二、再看右边：这些是常见解决办法

### 1. 梯度裁剪

- 英文叫 `gradient clipping`
- 作用主要是处理梯度爆炸

它的思路很简单：

- 如果梯度太大，就不要让它按原来的大小更新
- 而是把它“截断”到一个可控范围内

比如：

- 本来梯度范数是 `100`
- 设定阈值是 `5`
- 那就把它缩到 `5` 附近再更新

它的核心作用：

- 防止参数一步更新太猛
- 防止训练突然发散
- 在 RNN 或较深网络里尤其常见

要注意：

- 梯度裁剪主要是救“爆炸”
- 它不能根本解决“梯度消失”

### 2. 残差网络

- 英文一般是 `ResNet` 或 `skip connection`
- 作用主要是缓解深层网络训练困难，尤其和梯度消失、退化问题有关

它的关键思想是：

- 不强迫每一层都去学一个完整的新映射
- 而是学“在原来基础上改多少”

公式上常写成：

- `y = F(x) + x`

也就是：

- 一条主路学新东西
- 一条捷径直接把原输入传过去

这样做的好处是：

- 梯度可以沿着捷径更容易传回前面层
- 深层网络更容易训练
- 不容易出现“层数加深反而效果更差”的退化问题

### 3. 权重初始化

- 就是网络一开始的参数不能乱设
- 如果初始值太大或太小，训练一开始就可能不稳定

为什么重要：

- 初始化太大，前向输出和反向梯度都可能越来越大，导致梯度爆炸
- 初始化太小，信号传着传着会越来越弱，导致梯度消失

常见思路：

- Xavier 初始化：常用于比较对称的激活条件
- He 初始化：常用于 ReLU 这类激活函数

本质上它们都在做一件事：

- 让每层输出的尺度别离谱
- 让前向和反向的数值尽量稳定

### 4. 归一化

- 常见的是 `BatchNorm`、`LayerNorm`
- 作用是让中间特征分布更稳定

直观理解：

- 如果每层输入分布老在剧烈变化，后面层就很难适应
- 归一化相当于把数据拉回一个更稳定的范围

它带来的好处通常有：

- 训练更稳定
- 可以用更大的学习率
- 收敛更快
- 某些情况下也能缓解梯度消失或爆炸

在不同模型里常见形式不同：

- CNN 里常见 BatchNorm
- Transformer 里更常见 LayerNorm

### 5. 动量法

- 英文是 `Momentum`
- 它是对普通梯度下降的改进

普通梯度下降的问题是：

- 每一步只看当前梯度
- 如果地形弯弯曲曲，就容易左右摇摆

动量法的思路是：

- 不只看当前这一步
- 还把前几步的“惯性”积累起来

你可以把它理解成：

- 一个小球从山坡往下滚
- 不只是当前坡度决定方向
- 还会带着之前滚动的惯性继续前进

它的优点：

- 减少来回震荡
- 沿正确方向前进得更快
- 提高收敛速度

### 6. RMSProp

- RMSProp 是一种自适应学习率方法

它解决的问题是：

- 不同参数对 loss 的敏感程度不一样
- 如果所有参数都用同一个学习率，有的可能太大，有的可能太小

RMSProp 的核心想法：

- 记录过去梯度平方的滑动平均
- 梯度大的方向，就把学习率适当调小
- 梯度小的方向，就别让更新太弱

你可以先记一句最实用的话：

- RMSProp 会“按参数维度自动调节步长”

### 7. Adam

- Adam 可以看成是 `Momentum + RMSProp` 的结合版

它同时做两件事：

- 用一阶矩估计保存“动量”
- 用二阶矩估计保存“梯度平方的历史信息”

所以 Adam 的优点通常是：

- 收敛快
- 对学习率没那么敏感
- 在很多任务上开箱即用

这也是为什么很多人训练网络时会先默认试 Adam。

不过也要注意：

- Adam 不一定总是最终泛化最好
- 有些任务最后还是 SGD + Momentum 更强

### 8. mini-batch

- mini-batch 就是每次不拿全部数据算梯度
- 而是拿一小批数据来估计梯度

这件事很重要，因为它同时影响：

- 训练速度
- 显存占用
- 梯度稳定性

三种常见方式：

- Batch Gradient Descent：每次用全数据
- Stochastic Gradient Descent：每次只用 1 个样本
- Mini-batch Gradient Descent：每次用一小批样本

mini-batch 为什么最常用：

- 比单样本更稳定
- 比全量数据更省时间
- 也更适合 GPU 并行计算

所以它实际上是在“稳定性”和“效率”之间找平衡。

## 三、这些方法和左边问题怎么对应

可以先记下面这张对应关系：

- 梯度消失：残差网络、权重初始化、归一化
- 梯度爆炸：梯度裁剪、权重初始化、归一化
- 收敛速度慢：动量法、RMSProp、Adam、归一化
- 计算开销大：mini-batch

要注意这里不是一一对应，而是：

- 一个问题可能有多个办法缓解
- 一个办法也可能同时改善多个问题

比如：

- 归一化既能让训练更稳，也常常能让收敛更快
- 权重初始化既影响梯度消失，也影响梯度爆炸
- mini-batch 不直接解决梯度问题，但能让训练更现实、更高效

## 四、把这一页串成一句人话

这页真正想表达的是：

- 神经网络训练不只是“算出梯度然后更新参数”这么简单
- 真正困难的是：梯度可能太小、太大，训练可能太慢，算力可能不够
- 所以后来大家发明了很多训练技巧，分别去解决这些现实问题

也就是说，第三节的重点不只是“怎么更新参数”，而是：

- 怎么让参数更新这件事变得稳定、有效、可落地

你原来已经记了这些内容：

- 泛化能力
- 过拟合
- 数据增强
- 正则化

- 超参数

这一页可以直接接在后面，形成完整逻辑：

### 第三节完整脉络

- 先学会怎么用梯度下降训练参数
- 再发现训练过程中会遇到很多问题
- 一类问题是泛化不好，也就是过拟合
- 另一类问题是训练本身不稳定或太慢
- 所以需要数据增强、正则化、dropout 去改善泛化
- 也需要梯度裁剪、残差网络、初始化、归一化、优化器、mini-batch 去改善训练过程

## 六、最适合现在记住的版本

如果你只想先抓重点，可以先记下面这些：

- 梯度消失：梯度传到前面层时太小，前面层学不动
- 梯度爆炸：梯度太大，参数更新失控
- 收敛速度：loss 下降快不快
- 计算开销：训练花多少时间和显存
- 梯度裁剪：防梯度爆炸
- 残差网络：让深层网络更好训练，缓解梯度消失
- 权重初始化：让训练一开始别崩
- 归一化：让特征分布更稳定，训练更快更稳
- 动量法：减少震荡，加快下降
- RMSProp：按参数自动调学习率
- Adam：动量法 + RMSProp
- mini-batch：每次用一小批数据训练，兼顾效率和稳定

## 七、你现在最容易混淆的点

### 1. 梯度消失和过拟合不是一回事

- 梯度消失：训练都训不动
- 过拟合：训练集学得太好，测试集不行

### 2. Adam 和学习率不是替代关系

- Adam 是优化器
- 学习率还是要设
- 只是 Adam 会帮你更智能地调不同参数的步长

### 3. mini-batch 不是“为了效果更好”才存在

- 它首先是为了让训练可算
- 然后顺便在效率和稳定性之间取得一个平衡

## 八、一句话总结

第三节结尾这页，其实是在补一句很重要的话：

- 神经网络不是知道“梯度下降”就够了，真正把网络训好，还要处理梯度稳定性、收敛效率和计算资源这三大现实问题。
