## 一、这次学习的核心目标

这次学习的目标，不只是把视频看完，而是建立一条完整的知识链条：

`函数 -> 神经网络 -> CNN -> RNN/Transformer -> ViT -> 多模态 RGB-D 任务`

## 二、系列视频学习笔记

### 1. 从函数到神经网络

这一节最核心的作用，是把“神经网络”从一个抽象名词变回“函数逼近器”。

我的理解是：

- 机器学习本质上是在学一个从输入到输出的映射函数。
- 最简单的模型可以看成一个线性函数，但线性函数表达能力有限。
- 神经网络通过多层线性变换和非线性激活的叠加，把简单函数组合成复杂函数。
- 层数越深，模型可以表达的映射关系越复杂，因此更适合处理图像、文本等高维数据。

这部分对我最大的帮助是，它让我重新理解了“编码器”和“解码器”。

- 编码器：把原始输入逐步映射到更抽象、更可分的特征空间。
- 解码器：把高层语义特征还原成任务需要的输出形式。

对于我的课题来说，语义分割网络中的 backbone 本质上就是编码器，后面的 FPN 或 segmentation head 就是解码器。

### 2. 计算神经网络的参数

这一节主要帮助我把“网络结构”转化成“可以被计算和训练的参数系统”。

我理解到：

- 每一层神经网络本质上都对应权重和偏置。
- 输入维度和输出维度一旦确定，这层的参数量也就能算出来。
- 神经网络不是一个神秘黑盒，而是由很多矩阵乘法和逐元素非线性操作构成。
- 参数量越大，模型表达能力一般越强，但同时训练成本和过拟合风险也会上升。

这部分和我现在做实验很相关，因为后面无论是加模块还是换 backbone，都不能只看“效果好不好”，还要看：

- 参数量有没有明显增加
- 计算量是否过大
- 显存是否能够承受

老师之前也提到过 8GB 显存的现实约束，所以后面我做模型改进时，不能只追求复杂度，还要考虑可训练性。

### 3. 调教神经网络的方法

这一节更偏训练方法论，对我后面跑 baseline 很有指导意义。

我总结出的重点有：

- 优化器决定参数沿什么方向更新。
- 学习率决定每次更新走多大步。
- 学习率过大容易震荡，过小又会训练太慢甚至停在较差位置。
- 训练过程中要关注过拟合问题，因此验证集指标和 early stopping 很重要。
- checkpoint 的作用不是“存档”，而是保留训练过程中表现最好的模型。

这和老师上次讲的内容基本一致。老师在笔记里也提到了：

- optimizer
- learning rate
- checkpoint
- early stopping

这说明老师并不是只关注模型结构本身，也很看重训练细节。后面如果我真的要写论文，训练配置、早停策略、最优 checkpoint 选择，这些都必须写清楚。

### 4. 从矩阵到 CNN

这一节帮助我把卷积神经网络和图像结构真正对应起来。

我目前的理解是：

- 图像本质上可以看成矩阵或张量。
- CNN 之所以适合图像，是因为它利用了图像的局部性和空间结构。
- 卷积核只看局部区域，相当于在图像上滑动提取局部模式。
- 参数共享使 CNN 在处理图像时比全连接层更高效。
- channel 和 spatial 是图像特征中的两个核心维度。

这一节和老师之前写下的关键词也能对应上：

- channel：通道数
- spatial：图像空间位置与局部结构

这让我更清楚地意识到，CNN 的优势来自很强的图像归纳偏置：它天然假设相邻像素相关、局部模式重要。这也是后面理解 ViT 为什么和 CNN 不一样的前提。

### 5. 从 RNN 到 Transformer

这一节最大的价值，是让我明白 Transformer 不是“凭空冒出来的”，而是为了解决序列建模里的老问题。

我的理解是：

- RNN 适合处理序列，因为它按时间顺序逐步读取输入。
- 但 RNN 存在长程依赖难建模、并行效率低等问题。
- 当序列变长时，前面的信息传到后面会逐渐衰减。
- Transformer 的突破点是：不再强依赖递归，而是让每个位置直接和其他位置建立联系。

所以 Transformer 的关键不只是“结构新”，而是它改变了处理序列的方式：

- RNN：一步一步传
- Transformer：全局两两交互

这也解释了为什么 attention 机制后来会成为核心。

### 6. Transformer 简单而强大

这一节是整个系列里和我当前课题最相关的一节。

我目前对 Transformer 的理解可以概括成下面几句话：

- 输入先被映射成向量表示。
- 每个位置都会生成 query、key、value。
- query 和所有 key 做匹配，得到相关性权重。
- 再用这些权重对 value 加权求和，得到新的表示。

这就是 self-attention 的核心逻辑：当前 token 不只是看自己，而是根据相关性去聚合其他 token 的信息。

我觉得它强大的原因主要有三点：

- 能直接建模长距离依赖
- 所有位置可以并行计算
- 不再局限于局部感受野

但它也不是没有代价：

- 注意力计算通常是两两关系，复杂度较高
- 缺少 CNN 那种天然的图像归纳偏置

这也正好能引出老师让我要去看的 ViT 论文。

## 三、结合老师要求补充的理解

### 1. attention 机制

老师明确要求我补 attention 机制，这说明后面的模型改进很可能和“融合方式”有关，而不是只换一个普通卷积模块。

我现在对 attention 的理解是：

- attention 的本质不是“看重点”这么简单，而是“基于相关性分配信息聚合权重”。
- self-attention 是同一个序列内部做相关性建模。
- cross-attention 则可以用于不同模态、不同序列之间的信息交互。

如果把这个思路放到 RGB-D 多模态任务里，就很自然会想到：

- RGB 特征和 depth 特征能否通过注意力机制进行更有针对性的交互
- depth 信息是否不仅仅是第二路输入，而是可以作为先验去引导 RGB 特征提取

这其实已经很接近论文改进点的思路了。

### 2. Transformer 原理

Transformer 一般可以概括为以下几个核心组件：

- 输入嵌入
- 位置编码
- 多头自注意力
- 前馈网络
- 残差连接和层归一化

我现在觉得最重要的不是死记结构图，而是理解它在做什么：

- 通过注意力建立全局依赖
- 通过多头机制从不同子空间提取关系
- 通过前馈网络进一步进行非线性变换
- 通过位置编码弥补“没有序列顺序感”的问题

### 3. ViT 的基本思想

ViT 论文《An Image is Worth 16x16 Words》给我的核心启发是：

- 图像也可以像文本一样，被切成 patch token
- 每个 patch 可以看成视觉领域里的“词”
- patch 序列加上位置编码后，可以直接送进 Transformer encoder

它和 CNN 最大的不同在于：

- CNN 靠局部卷积提特征
- ViT 靠 patch token 之间的全局注意力建模关系

所以老师说“了解 ViT”，我理解重点不在于把论文所有公式都背下来，而是要搞懂：

- 图像为什么可以切 patch
- patch embedding 是什么
- ViT 为什么不再依赖卷积
- ViT 的优势和局限各是什么

### 4. ViT 和 CNN 的关系

我目前的理解不是“ViT 完全替代 CNN”，而是：

- CNN 更擅长利用局部结构和归纳偏置
- ViT 更擅长全局依赖建模和大规模预训练迁移
- 在很多视觉任务里，二者可以竞争，也可以结合

这对我后续选方向很重要。因为我的任务不是纯图像分类，而是 RGB-D 语义分割。分割任务既需要局部边缘和细节，又需要全局语义理解，因此后续不一定要极端地只选 CNN 或只选 Transformer，更合理的思路是看怎么利用二者各自的优点。

## 四、和当前多模态课题的联系

结合老师之前给的笔记和现在的代码框架，我对当前课题的定位更清楚了。

### 1. 任务本质

我当前做的是 RGB-D 多模态语义分割。

- RGB 提供颜色、纹理、语义外观信息
- Depth 提供几何、距离、空间结构信息

这两者天然就是多模态。

老师笔记里提到“rgb 和 depth 也是多模态”，这句话虽然简单，但很关键。它意味着后面做改进时不能只把 depth 当成“多一张图”，而要真正理解它提供的是不同性质的信息。

### 2. 当前框架能做什么

课程代码框架已经给了一个能运行的研究骨架，大概包括：

- early fusion
- mid fusion
- attention fusion
- advanced(ViT 预训练骨干) 的思路说明

它的意义更像是：

- 帮我把数据、训练、评估、可视化全部打通
- 给我一个可以做 ablation 的基本平台
- 让我先有一个能跑的 baseline，再去谈改进

所以它适合作为我自己的工程 baseline，但未必就是后面论文里的最终“强基线论文”。

### 3. 老师对论文工作的实际要求

从之前的交流和笔记看，老师对后面论文工作的要求至少包括：

- 先有一个能跑通的 baseline

这说明后面真正写论文时，结构上很可能会是：

- 消融实验

## 五、现阶段我学到的东西

这次学习后，我觉得自己至少有下面几个认识比之前更清楚了。

第一，神经网络不是一个突然出现的黑箱，而是一条逐步发展起来的技术链条：从函数拟合，到多层网络，到卷积网络，再到序列建模和 Transformer。

第二，CNN 和 Transformer 并不是简单的“谁先进谁落后”的关系，而是各自抓住了不同的数据结构特征。CNN 更强在局部归纳偏置，Transformer 更强在全局关系建模。

第三，attention 机制对我后面的多模态研究特别重要。因为多模态融合的关键问题，本质上就是“不同来源的信息如何建立联系并分配权重”。

第四，ViT 的意义不只是提出一个新模型，而是把图像也转化成 token 序列来处理，这种思想对理解后面的视觉 Transformer、分割 Transformer、以及多模态 Transformer 都很重要。

第五，老师现在让我补这些基础，并不是为了考试式背概念，而是为了让我后面在自己的 RGB-D baseline 上能看懂改进点、设计出合理模块，并写出像样的学习报告和论文。
