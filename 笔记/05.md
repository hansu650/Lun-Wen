# 05 注意力融合篇整理

## 这一章在讲什么

这一章的核心不是“把模型做得更大”，而是回答一个更细的问题：

- RGB 和 Depth 到底应该怎么更精细地融合？
- 为什么只用一个统一门控 `g` 还不够？
- 能不能分别在“通道维度”和“空间维度”上决定谁更重要？

如果用一句话概括，本章讲的是：

> 从 `mid fusion` 继续往前走，把“是否融合、怎么融合”做得更细，而不是只做一个粗粒度加权。

---

## 1. 为什么从门控融合继续走到注意力融合

你前面已经学过 `mid_fusion` 的基本想法：

- RGB 先提 RGB 特征
- Depth 先提 Depth 特征
- 中间层做融合

课程里先用 `Gated Fusion` 做了一个很自然的版本：

```python
fused = g * rgb_feat + (1 - g) * depth_feat
```

这个式子直观，但它有一个局限：

- `g` 虽然能控制两种模态的比例
- 但它不够“细”
- 现实里有些信息更应该按通道选，有些信息更应该按空间位置选

举两个直觉例子：

- 颜色、纹理相关的信息，通常 RGB 更强
- 边缘、遮挡边界、距离结构，很多时候 Depth 更强

所以课程 05 的思路是：

- 不再只用一个粗粒度的融合权重
- 而是把“重要性”拆开看
- 让模型自己学：
  - 哪些通道更重要
  - 哪些空间位置更重要

---

## 2. 这一章的两个核心：通道注意力 + 空间注意力

### 2.1 通道注意力（Channel Attention）

通道注意力回答的问题是：

> 在语义层面上，哪些特征通道更值得相信？

课程里的标准流程是：

```python
channel_weights = sigmoid(mlp(gap(concat([rgb, depth]))))
```

可以拆成 3 步理解：

1. 先把 RGB 和 Depth 特征拼起来
2. 对每个通道做全局平均池化（GAP）
3. 再用轻量 MLP/1x1 Conv 学通道权重

它的特点是：

- 参数量小
- 计算开销低
- 更偏“全局语义判断”

你可以把它想成：

- 这一层里有些通道是在表达纹理
- 有些通道是在表达边缘
- 有些通道是在表达室内布局
- 模型会学着给它们不同的重要性

### 2.2 空间注意力（Spatial Attention）

空间注意力回答的问题是：

> 在图像平面上，哪些位置更值得关注？

课程里的标准形式是：

```python
spatial_weights = sigmoid(conv(concat([rgb, depth])))
```

它更关注：

- 物体边界
- 显著区域
- 遮挡过渡区域
- 远近结构变化明显的位置

如果说通道注意力在回答“看什么特征”，  
那空间注意力更像在回答“看图上的哪里”。

## 3. 课程里的 Cross-Modal Attention Fusion 在做什么

这一章的融合模块不是完整 Transformer 自注意力，而是一个比较轻量、适合教学和工程入门的设计。

它大致分 5 步：

1. **投影对齐**

- 先把 RGB 特征和 Depth 特征投影到同样的通道数
- 否则后面不好直接比较和融合

2. **通道注意力**

- 对拼接后的特征做全局统计
- 给每个通道一个权重

3. **空间注意力**

- 对拼接后的特征做轻量卷积
- 给每个空间位置一个权重

4. **特征交互**

- 把加权后的两种模态继续做卷积交互
- 不是只停留在简单加权相加

5. **残差连接**

- 保证训练更稳
- 防止全靠注意力分支导致梯度传递太脆弱

课程里的完整想法可以压缩成：

```text
RGB feat / Depth feat
-> project to same channels
-> channel attention
-> spatial attention
-> interaction conv
-> residual add
-> fused feature
```

## 4. 为什么课程里强调“高效注意力”

这一章其实有一个很重要的工程意识：

> 不是所有“注意力”都值得上。

课程明确提醒要避免：

- 构造完整的 `[HW, HW]` 空间自注意力矩阵
- 过大的卷积核
- 过多的中间通道

原因很直接：

- 语义分割输入分辨率高
- `H * W` 很大
- 一旦做全局空间自注意力，显存和计算量会暴涨

所以课程 05 推的是：

- 通道注意力
- 轻量空间注意力
- 分组卷积 / 深度可分离卷积

这章真正想训练你的，不只是“会写模块”，还有：

> 做 RGB-D 分割时，要一直有“精度 vs 计算代价”的意识。

## 6. 这一章你最该真正学会什么

如果按“课程理解”来排优先级，我觉得本章最重要的是这 4 件事：

### 第一层：概念层

- 注意力融合和门控融合的差别是什么
- 为什么要区分通道维和空间维

### 第二层：模块层

- 一个轻量跨模态注意力模块通常怎么搭
- 为什么要有投影对齐、交互卷积和残差

### 第三层：工程层

- 为什么不能一上来就上全局自注意力
- 高分辨率分割任务里，复杂注意力很容易代价过大

### 第四层：实验层

- 只做通道注意力 vs 只做空间注意力 vs 两者结合
- 这类消融实验才真正说明模块为什么有效

## 8. 这章暂时不要误解的地方

### 不是“注意力 = 一定更强”

注意力模块可能提升精度，但也可能：

- 参数变多
- 训练变慢
- 对小数据集更容易不稳定

### 不是“课程里讲了 = 本地代码里一定有”

你现在这份骨架仍然只该先聚焦：

- `early`
- `mid_fusion`
- `train / eval / infer`

课程 05 更适合你当成：

- 理解后续研究方向
- 为以后自己加模块做准备

如果把课程 05 压缩成一段你以后能反复回看的话，就是：

> 门控融合已经能做“是否更信 RGB 或 Depth”的粗粒度判断，但注意力融合把这个判断拆到了更细的维度上：通道维度决定“看什么特征”，空间维度决定“看图上的哪里”。课程 05 的重点不是炫复杂模块，而是学会在“更细粒度融合”和“计算效率”之间做平衡。
