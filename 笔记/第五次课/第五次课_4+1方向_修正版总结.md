# 第五次课 4+1 方向修正版总结

> 本笔记用于整理第五次课网站 Section 2 的学习脉络，并补充我对 RGB-D 4+1 方向的理解。  
> 这里不是论文摘要，也不是已经完成的实验结论；主要目的是把“老师说的方向”“网站里的公式”和“我当前 baseline 的下一步判断”放到一条清楚的笔记线里。

## 0. 这份笔记和原网站 Section 2 的关系

原网站 Section 2 的主线是：

```text
背景 -> 问题 -> 改进方向 -> 预期收益 -> 参考文献索引
```

它一直在强调一件事：不要先记模块名，而要先问这个模块在修复什么失败模式。

对应到我的 RGB-D 语义分割任务，现在最重要的不是继续堆更复杂的 fusion block，而是先把问题拆开：

- 边界和细结构丢失：对应频域、多分辨率、高频增强。
- RGB 和 Depth 互相干扰：对应映射、对齐、筛选、协方差约束。
- 训练只靠最终 CE loss：对应辅助 loss、自监督、重建、对比学习。
- 主干很强但适配不稳：对应轻量 head、adapter、frequency residual branch。

所以我对这次 4+1 的理解是：

```text
4 = 更偏结构 / 表征 / encoder 的方向
+1 = 更偏辅助 loss / 训练约束的方向
```

最核心的区分：

```text
encoder 频域增强 = 改结构 / 改特征提取
+1 loss = 辅助监督 / 训练约束
head = 为 loss 提供可计算输出
loss 会通过反向传播约束 encoder / depth encoder / fusion module / decoder feature
```

## 1. 老师整体想表达什么

这次老师说的 RGB-D 4+1，不是让我继续在 baseline 后面堆很多 fusion module。

我现在的主线 baseline 是：

```text
DFormerV2_S + DepthEncoder + GatedFusion + SimpleFPNDecoder
```

模型名：

```text
dformerv2_mid_fusion
```

之前自研 fusion 模块多次重复后没有稳定超过这个 baseline，所以这次重点应该从：

```text
fusion module design
```

往前移动到：

```text
encoder 表征质量
+ 频域结构
+ 跨模态统计关系
+ 辅助监督信号
```

也就是说，不是继续问：

```text
RGB feature 和 Depth feature 已经出来了，我再怎么融合？
```

而是先问：

```text
RGB 和 Depth 在进入 fusion 之前，是否已经被正确提取、对齐、增强和约束？
```

## 2. 当前 baseline 的基础公式

设 RGB 输入为：

$$
X_r \in \mathbb{R}^{3 \times H \times W}
$$

Depth 输入为：

$$
X_d \in \mathbb{R}^{1 \times H \times W}
$$

经过两个 encoder 后，第 $i$ 个 stage 的特征可以写成：

$$
F_r^i = E_r^i(X_r)
$$

$$
F_d^i = E_d^i(X_d)
$$

其中：

- $F_r^i$ 是 RGB encoder 第 $i$ 层特征。
- $F_d^i$ 是 Depth encoder 第 $i$ 层特征。
- $i \in \{1,2,3,4\}$，对应多尺度特征层。

一个抽象的融合函数可以写成：

$$
F_f^i = \phi(F_r^i, F_d^i)
$$

其中 $\phi(\cdot)$ 可以是 concat、add、gate、attention 等融合方式。

### 2.1 GatedFusion 公式

当前 baseline 里的 GatedFusion 可以用下面的概念公式理解：

$$
G^i = \sigma\left(\mathrm{Conv}\left([F_r^i, F_d^i]\right)\right)
$$

$$
F_f^i = G^i \odot F_r^i + (1 - G^i) \odot F_d^i
$$

变量解释：

- $[F_r^i, F_d^i]$ 表示通道拼接。
- $\sigma(\cdot)$ 是 sigmoid，把 gate 限制到 $0$ 到 $1$。
- $G^i$ 是门控权重。
- $\odot$ 是逐元素相乘。
- $F_f^i$ 是融合后的第 $i$ 层特征。

直观理解：

```text
模型自己学习每个位置、每个通道更应该相信 RGB 还是 Depth。
```

但这里有一个问题：如果 RGB 和 Depth 本身没有对齐，或者 Depth feature 质量不好，那么 gate 学到的权重也可能不稳定。

这就是为什么老师这次强调的方向不只是“换 fusion”，而是要往 encoder 表征、频域增强、对齐约束和辅助监督上看。

## 3. 结构方向一：映射、对齐、筛选

### 3.1 为什么要先 projection

RGB 和 Depth 来自不同模态，原始特征不一定在同一个空间中可比较。

所以可以先通过 projection head 把两者映射到共享空间：

$$
Z_r^i = P_r(F_r^i)
$$

$$
Z_d^i = P_d(F_d^i)
$$

其中：

- $P_r(\cdot)$ 是 RGB projection head。
- $P_d(\cdot)$ 是 Depth projection head。
- $Z_r^i$ 和 $Z_d^i$ 是映射后的可比较特征。

最轻量的 projection 可以是 $1 \times 1$ convolution：

$$
P_r(F_r^i) = \mathrm{Conv}_{1 \times 1}(F_r^i)
$$

$$
P_d(F_d^i) = \mathrm{Conv}_{1 \times 1}(F_d^i)
$$

这和网站里“共享子空间 / 先对齐后融合”的思想一致：融合之前，先让两种模态落到一个更容易比较的表示空间。

### 3.2 对齐目标

一个朴素的对齐 loss 可以写成：

$$
\mathcal{L}_{align}^i = 1 - \cos(Z_r^i, Z_d^i)
$$

其中 cosine similarity 可以理解为：

$$
\cos(Z_r^i, Z_d^i)
=
\frac{\langle Z_r^i, Z_d^i \rangle}
{\|Z_r^i\|_2 \|Z_d^i\|_2}
$$

如果两个模态在同一位置表示的是同一个物体或区域，那么投影后的特征应该更接近。

### 3.3 筛选可靠信息

还可以学习一个可靠性权重：

$$
R^i = \sigma\left(\mathrm{Conv}\left([Z_r^i, Z_d^i, |Z_r^i - Z_d^i|]\right)\right)
$$

再用它筛选 Depth 信息：

$$
\tilde{F}_d^i = R^i \odot F_d^i
$$

$$
F_f^i = \phi(F_r^i, \tilde{F}_d^i)
$$

直观理解：

```text
先映射，再对齐，再筛选，最后融合。
```

这个方向属于结构 / 表征方向，不是单独的 loss 方向。但如果加上 $\mathcal{L}_{align}$，就会进入辅助监督。

## 4. 结构方向二：encoder 频域增强

老师说的“Depth 傅里叶 / 卷积 / 可以改 depth backbone”，我现在理解为：

```text
从 RGB 或 Depth 原图 / 特征中提取频域表示
-> 做平滑、增强、高频/低频分解
-> 还原回空间域
-> residual 加回 encoder 或 depth encoder
```

核心形式可以写成：

$$
x_{enhanced} = x + \mathrm{FrequencyEnhance}(x)
$$

这里 $x$ 可以是原图，也可以是 encoder 中间特征。

### 4.1 高频 / 低频分解公式

网站里反复出现的高低频分解可以写成：

$$
f = f_{low} + f_{high}
$$

$$
f_{low} = G_{\sigma} * f
$$

$$
f_{high} = f - G_{\sigma} * f
$$

变量解释：

- $f$ 是原图或特征图。
- $G_{\sigma}$ 是高斯平滑核。
- $*$ 表示卷积。
- $f_{low}$ 表示低频，偏整体轮廓、平滑主体、场景布局。
- $f_{high}$ 表示高频，偏边界、细纹理、轮廓突变和细物体。

直观理解：

```text
先用平滑得到低频；
再用原图减去平滑结果得到高频。
```

### 4.2 Depth Encoder Frequency Residual Enhancement

这是我认为可以作为结构候选 B 的方向。

对 Depth feature 做分解：

$$
F_{d,low} = G_{\sigma} * F_d
$$

$$
F_{d,high} = F_d - F_{d,low}
$$

分别增强：

$$
\hat{F}_{d,low} = \psi_{low}(F_{d,low})
$$

$$
\hat{F}_{d,high} = \psi_{high}(F_{d,high})
$$

再组合：

$$
F_{d,freq} = \hat{F}_{d,low} + \gamma \hat{F}_{d,high}
$$

最后 residual 加回 depth encoder：

$$
F_{d,out} = F_d + \alpha \cdot F_{d,freq}
$$

变量解释：

- $\psi_{low}$ 可以是普通卷积，用来稳定主体结构。
- $\psi_{high}$ 可以是边界增强卷积，用来突出轮廓和深度变化。
- $\gamma$ 控制高频增强强度。
- $\alpha$ 控制整个 frequency residual 的强度。

这条路线属于：

```text
改 encoder / 改 depth backbone / 改 feature extraction
```

不是 loss。

原因很简单：它改变了 forward 里的特征计算路径，推理时也会存在。

## 5. 结构方向三：RGB 高频特征与 Depth 几何比较

RGB 高频通常包含纹理、边缘、细节；Depth 高频通常包含几何边界、前后层次、深度突变。

所以可以分别提取：

$$
F_{r,high} = F_r - G_{\sigma} * F_r
$$

$$
F_{d,high} = F_d - G_{\sigma} * F_d
$$

再比较两者差异：

$$
D_{high} = \left|P_r(F_{r,high}) - P_d(F_{d,high})\right|
$$

然后生成一个残差补偿：

$$
\Delta F = \psi(D_{high}, F_{r,high}, F_{d,high})
$$

最后加回融合特征：

$$
F_f = F_f + \beta \Delta F
$$

变量解释：

- $P_r$ 和 $P_d$ 用来把 RGB 高频和 Depth 高频映射到同一维度。
- $D_{high}$ 表示两个模态高频结构的差异。
- $\Delta F$ 是根据高频差异生成的补偿。
- $\beta$ 控制补偿强度。

直观理解：

```text
如果 RGB 边界清楚，但 Depth 边界模糊，可以用 RGB 高频补；
如果 Depth 几何边界可靠，但 RGB 纹理干扰大，就不能盲目相信 RGB 高频。
```

这个方向比随便加 residual 更合理，因为 residual 来自 RGB 高频与 Depth 几何结构的比较。

## 6. +1：辅助 loss / 自监督 / 训练约束

老师说的 `+1` 可以理解成一组辅助监督，不是单独一个普通 loss。

主任务 loss 仍然是语义分割：

$$
\mathcal{L}_{seg} = \mathrm{CE}(Y, \hat{Y})
$$

其中：

- $Y$ 是真实标签。
- $\hat{Y}$ 是模型预测的 segmentation map。

如果加入辅助 loss，一个总形式可以写成：

$$
\mathcal{L}_{total}
=
\mathcal{L}_{seg}
+ \lambda_{freq}\mathcal{L}_{freq-cov}
+ \lambda_{mask}\mathcal{L}_{mask}
+ \lambda_{rec}\mathcal{L}_{rec-cov}
+ \lambda_{contrast}\mathcal{L}_{contrast}
$$

这里的重点不是第一版全部加上，而是说明这些 loss 属于不同监督信号：

- $\mathcal{L}_{freq-cov}$：高频 / 低频统计关系。
- $\mathcal{L}_{mask}$：掩码跨模态重建。
- $\mathcal{L}_{rec-cov}$：重建结果的统计关系。
- $\mathcal{L}_{contrast}$：RGB / Depth 对比式对齐。

第一版实验不能一次全加。

## 7. Loss 一：高频 / 低频协方差 loss

这个方向可以叫：

```text
Frequency Covariance Loss
```

先分解 RGB 和 Depth 特征：

$$
F_{r,low} = G_{\sigma} * F_r
$$

$$
F_{r,high} = F_r - F_{r,low}
$$

$$
F_{d,low} = G_{\sigma} * F_d
$$

$$
F_{d,high} = F_d - F_{d,low}
$$

通过 projection head 映射：

$$
Z_{r,high} = P_{r,high}(F_{r,high})
$$

$$
Z_{d,high} = P_{d,high}(F_{d,high})
$$

$$
Z_{r,low} = P_{r,low}(F_{r,low})
$$

$$
Z_{d,low} = P_{d,low}(F_{d,low})
$$

### 7.1 协方差矩阵

对一个展平后的特征矩阵 $Z \in \mathbb{R}^{N \times C}$，其中 $N = H \times W$，$C$ 是通道数。

先中心化：

$$
\bar{Z} = Z - \frac{1}{N}\sum_{n=1}^{N} Z_n
$$

协方差矩阵为：

$$
\mathrm{Cov}(Z) = \frac{1}{N-1}\bar{Z}^{T}\bar{Z}
$$

### 7.2 Frequency Covariance Loss 公式

高频协方差约束：

$$
\mathcal{L}_{high-cov}
=
\left\|
\mathrm{Cov}(Z_{r,high})
-
\mathrm{Cov}(Z_{d,high})
\right\|_F^2
$$

低频协方差约束：

$$
\mathcal{L}_{low-cov}
=
\left\|
\mathrm{Cov}(Z_{r,low})
-
\mathrm{Cov}(Z_{d,low})
\right\|_F^2
$$

最终：

$$
\mathcal{L}_{freq-cov}
=
\mathcal{L}_{high-cov}
+ \eta \mathcal{L}_{low-cov}
$$

变量解释：

- $\|\cdot\|_F$ 是 Frobenius norm。
- $\eta$ 是低频项权重。
- 高频项更关注边界、轮廓、深度突变。
- 低频项更关注主体区域、场景布局。

我的理解：

```text
这个 loss 不是让高频和低频一样，
而是约束 RGB / Depth 在高频边界和低频主体上的统计关系。
```

这类 loss 不需要生成图像，只需要 projection head 或 $1 \times 1$ conv head。

## 8. Loss 二：掩码跨模态重建 loss

这个方向可以叫：

```text
Cross-modal Mask Reconstruction Loss
```

它的核心是：故意遮住一部分模态信息，再让模型用另一个模态补回来。

### 8.1 Mask RGB 猜 Depth

设 mask 后的 RGB 为：

$$
\tilde{X}_r = M_r \odot X_r
$$

其中：

- $M_r$ 是 RGB mask。
- $\tilde{X}_r$ 是被遮挡后的 RGB。
- $\odot$ 是逐元素乘法。

用 reconstruction head 预测 Depth：

$$
\hat{X}_d = R_d(\tilde{X}_r, X_d)
$$

重建 loss：

$$
\mathcal{L}_{rgb \rightarrow depth}
=
\left\|
(1 - M_r) \odot (\hat{X}_d - X_d)
\right\|_1
$$

这里只在被 mask 的区域计算 loss。

### 8.2 Mask Depth 猜 RGB

同理：

$$
\tilde{X}_d = M_d \odot X_d
$$

$$
\hat{X}_r = R_r(X_r, \tilde{X}_d)
$$

$$
\mathcal{L}_{depth \rightarrow rgb}
=
\left\|
(1 - M_d) \odot (\hat{X}_r - X_r)
\right\|_1
$$

但是 Depth 猜 RGB 更难，因为 Depth 没有颜色和纹理信息，所以 Depth 侧 mask 比例应该更小：

$$
\rho_d < \rho_r
$$

其中：

- $\rho_d$ 是 Depth mask ratio。
- $\rho_r$ 是 RGB mask ratio。

最终 mask reconstruction loss：

$$
\mathcal{L}_{mask}
=
\mathcal{L}_{rgb \rightarrow depth}
+ \mu \mathcal{L}_{depth \rightarrow rgb}
$$

变量解释：

- $\mu$ 控制 Depth 猜 RGB 这一项的权重。
- $R_d$、$R_r$ 是 reconstruction head 或 small decoder head。

这一类 loss 需要 reconstruction head / small decoder head。

## 9. Loss 三：重建式 covariance loss

掩码重建 loss 只问：

```text
生成结果像不像原图或原特征？
```

重建式 covariance loss 进一步问：

```text
生成结果和原始模态 / 跨模态之间的统计关系对不对？
```

例如先生成 Depth：

$$
\hat{X}_d = R_d(\tilde{X}_r, X_d)
$$

再提取特征：

$$
\hat{Z}_d = P_d(E_d(\hat{X}_d))
$$

$$
Z_d = P_d(E_d(X_d))
$$

重建式 covariance loss：

$$
\mathcal{L}_{rec-cov}
=
\left\|
\mathrm{Cov}(\hat{Z}_d)
-
\mathrm{Cov}(Z_d)
\right\|_F^2
$$

也可以做跨模态统计约束：

$$
\mathcal{L}_{cross-rec-cov}
=
\left\|
\mathrm{Cov}(\hat{Z}_d)
-
\mathrm{Cov}(Z_r)
\right\|_F^2
$$

这个方向可以概括为：

```text
先生成，再算统计关系。
```

它需要：

- reconstruction head；
- projection head；
- covariance 计算。

变量最多，所以不适合第一版一次加入。

## 10. Loss 四：对比式 covariance / InfoNCE loss

如果不做生成，可以直接把 RGB feature 和 Depth feature 输入 projection head，然后做对比学习。

对同一样本、同一位置或同一语义区域：

$$
z_r = P_r(F_r)
$$

$$
z_d^+ = P_d(F_d)
$$

其中 $z_d^+$ 是与 $z_r$ 对应的正样本。

其他样本或其他位置的 Depth 特征可以作为负样本：

$$
z_{d,j}^{-}
$$

### 10.1 InfoNCE loss 公式

RGB 到 Depth 的对比 loss：

$$
\mathcal{L}_{r \rightarrow d}
=
-
\log
\frac{
\exp(\mathrm{sim}(z_r, z_d^+) / \tau)
}{
\exp(\mathrm{sim}(z_r, z_d^+) / \tau)
+
\sum_j \exp(\mathrm{sim}(z_r, z_{d,j}^{-}) / \tau)
}
$$

Depth 到 RGB 的对比 loss：

$$
\mathcal{L}_{d \rightarrow r}
=
-
\log
\frac{
\exp(\mathrm{sim}(z_d, z_r^+) / \tau)
}{
\exp(\mathrm{sim}(z_d, z_r^+) / \tau)
+
\sum_j \exp(\mathrm{sim}(z_d, z_{r,j}^{-}) / \tau)
}
$$

最终：

$$
\mathcal{L}_{contrast}
=
\mathcal{L}_{r \rightarrow d}
+
\mathcal{L}_{d \rightarrow r}
$$

变量解释：

- $\mathrm{sim}(\cdot,\cdot)$ 可以是 cosine similarity。
- $\tau$ 是温度系数。
- 正样本应该更接近。
- 负样本应该更远。

老师提到 RGB 和 Depth 分母可以分开理解，我现在的理解是：负样本池设计要谨慎，不一定把所有东西都混成一个池子。

可以分成：

```text
RGB 内部负样本
Depth 内部负样本
跨模态负样本
```

这一点需要再和老师确认。

## 11. Head、loss 和反向传播关系

这里我最需要记住的是：

```text
head 不是最终目的；
head 是为了让某个 loss 有可计算的输出。
```

| 方向 | 需要的 head | 是否改变推理结构 | 主要会约束哪些模块 |
|---|---|---|---|
| 高频 / 低频协方差 loss | $1 \times 1$ projection head | 否 | encoder、depth encoder、projection head |
| 掩码跨模态重建 loss | reconstruction head / small decoder head | 训练时需要，推理可去掉 | encoder、depth encoder、reconstruction head |
| 重建式 covariance loss | reconstruction head + projection head | 训练时需要，推理可去掉 | encoder、depth encoder、reconstruction head、projection head |
| 对比式 InfoNCE loss | projection head | 否 | encoder、depth encoder、projection head |
| encoder 频域增强 | frequency branch / residual adapter | 是 | encoder 本身和 frequency branch |

更具体地说：

- loss 接在 fusion 前：主要约束 RGB encoder / Depth encoder 的表征。
- loss 接在 fusion 后：会约束 encoder、depth encoder 和 fusion module。
- loss 接在 decoder feature：还会进一步影响 decoder feature。
- reconstruction head 通常是训练辅助，推理时可以去掉。
- frequency branch 如果写进 encoder forward，就是推理结构的一部分。

所以同样叫“加 head”，含义不一样：

```text
projection head：让特征能算对齐 / covariance / InfoNCE。
reconstruction head：让特征能生成被 mask 的 RGB / Depth / feature。
frequency branch：不是 loss head，而是结构分支。
```

## 12. 和我之前失败的 fusion 模块有什么区别

之前的自研模块主要是：

```text
已有 RGB feature + 已有 Depth feature
-> 设计更复杂 fusion block
```

比如 attention fusion、CMX、CMNeXt、PrimKD 风格 primary-guided fusion，本质上还是在问：

```text
RGB 和 Depth 已经出来了，我怎么融合？
```

这次 4+1 更强调：

```text
先提升 feature extraction
再做频域分析
再做跨模态统计约束
最后再融合
```

也就是说，不是继续问：

```text
我还能怎么 fusion？
```

而是先问：

```text
RGB 和 Depth 是否真的对齐？
Depth 几何边界是否被提出来？
高频和低频是否被区分？
辅助 loss 是否约束了跨模态关系？
```

一句话总结：

```text
以前是 fusion block 叠加；
现在是 encoder 表征 + 频域结构 + 跨模态监督共同设计。
```

## 13. 第一版实验建议：不要一次全加

现在最稳的候选有两个。

### 13.1 候选 A：Feature-level Frequency Covariance Loss

结构：

```text
DFormerV2 + DepthEncoder + GatedFusion + SimpleFPNDecoder
+ frequency covariance auxiliary loss
```

特点：

- 不改 GatedFusion。
- 不改 decoder。
- 不改推理结构。
- 只加训练阶段的 auxiliary loss。
- 最适合和当前 baseline 做干净 ablation。

总 loss：

$$
\mathcal{L}_{total}
=
\mathcal{L}_{seg}
+ \lambda_{freq}\mathcal{L}_{freq-cov}
$$

第一版可以只在一个 stage 上做，比如 $c_3$ 或 $c_4$。

不要四层全加，避免变量太多。

### 13.2 候选 B：Depth Encoder Frequency Residual Enhancement

结构：

```text
Depth feature
-> low/high frequency decomposition
-> frequency enhancement
-> residual add back
-> GatedFusion
```

公式：

$$
F_{d,out} = F_d + \alpha \cdot \mathrm{FrequencyEnhance}(F_d)
$$

特点：

- 更像结构创新。
- 更接近老师说的“改 depth backbone”。
- 会改变推理结构。
- 如果有效，更容易写成模块。
- 但变量比候选 A 多，所以需要更谨慎做 ablation。

### 13.3 当前建议

如果目标是先做最干净验证：

```text
先做候选 A：Feature-level Frequency Covariance Loss
```

如果老师更强调结构创新：

```text
优先做候选 B：Depth Encoder Frequency Residual Enhancement
```

不要第一版就同时做 A + B。

否则结果不好分析。

## 14. 现在不适合一次全加的东西

第一版暂时不要同时加入：

- encoder 频域增强 branch。
- mask RGB 猜 Depth。
- mask Depth 猜 RGB。
- reconstruction-based covariance loss。
- 多尺度 frequency covariance。
- InfoNCE + reconstruction 联合训练。
- 新 fusion module。

原因不是这些方向不好，而是变量太多。

如果同时改 encoder、加 frequency branch、加 reconstruction、加 covariance、加 InfoNCE、再改 fusion，就算涨了也不知道是谁涨的；如果掉了也不知道是谁害的。

## 15. 需要和老师确认的点

下面这些点我先保留，不直接删，明天可以问老师确认：

1. **Frequency Covariance Loss 具体算什么 covariance**  
   是只比较 RGB 高频和 Depth 高频的 covariance，还是要算 high-low 之间的 cross-covariance？比如 $\mathrm{Cov}(Z_{high}, Z_{low})$ 这种形式。

2. **高频 / 低频分解用 FFT 还是卷积近似**  
   网站里用 $G_{\sigma} * f$ 解释更直观，但老师说“傅里叶”时，是否希望真的进入 FFT 频域再做滤波，需要确认。

3. **InfoNCE 的负样本池怎么分**  
   “RGB 和 Depth 分母分开，分两个池子”到底是 RGB 内部、Depth 内部各自做 denominator，还是跨模态正负样本分开采样，需要确认。

4. **Mask reconstruction 是重建图像还是重建 feature**  
   重建 RGB / Depth 原图更直观，但重建 feature 可能更稳、更省显存。这个实现层面需要问清楚。

5. **loss 接在 fusion 前还是 fusion 后**  
   如果接在 fusion 前，更容易说明它约束 encoder 表征；如果接在 fusion 后，会影响 fusion module，但分析更复杂。

6. **候选 A 和候选 B 的优先级**  
   如果老师更想要结构创新，候选 B 更贴近“改 depth backbone”；如果先要干净实验，候选 A 更适合。
