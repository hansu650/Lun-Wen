# 第二次课补充整理

以下内容是在上面课堂速记基础上的小幅整理，尽量只补充、不删减原意。

## 1. 这次课主要在讲什么

- 这次课的重点还是 RGB-D 语义分割，但关注点已经从“先把流程跑通”转到“下一步怎么把效果做上去”。
- 讨论的主线比较明确：**编码器/backbone、模态融合、解码器** 这三个位置分别能怎么改。
- 课堂上提到的一个结果目标是：如果方法想继续往下做，最好能把 `mIoU` 推到 `54` 以上，再考虑作为一个更正式的方向继续细化。

## 2. 编码器 / Backbone 这部分的理解

- 编码器通常是整套模型里最重的一部分，课堂上也提到“大头在 encoder”，参数量可能就在 `40M` 左右这个量级。
- 因此一个比较稳的策略不是一开始大改编码器，而是**尽量少改 backbone，本体尽量复用成熟预训练模型**。
- 课堂里提到的候选 backbone 方向包括：
  - `ViT`
  - `DINO` 系列 ViT
  - `SAM` 的 encoder
  - `Swin Transformer`
- 如果大模型直接跑不起来，就先退到 lite 版本，先验证思路，再考虑往大模型迁移。

## 3. VPT 微调思路可以怎么理解

- `VPT shallow`：更接近“只在最前面插入 prompt/token”，改动范围更小。
- `VPT deep`：在每一层都引入 prompt，属于更深一点的提示微调。
- 课上提到一个很值得记住的想法：**可以把 depth 信息做成新的、可学习且不冻结的 token，再送进 ViT 做微调**。
- 这个思路的好处是：不用完全重写 backbone，而是把深度模态的信息通过 token 形式注入进去。

## 4. 模态融合这块，老师强调了什么

- 融合不是简单拼接就结束，关键在于 **RGB 和 depth 的特征交互**。
- 课堂里提到的交互关键词主要有：
  - `cross attention`
  - `channel attention`
  - `spatial attention`
  - `depth-aware attention`
- 一个比较重要的实现约束是：**融合模块尽量保持输入输出维度不变**，这样更容易和前后结构兼容。
- “2块”可以先理解成：可以在网络中设计两处特征交互模块，先做少量关键位置的融合，不一定一开始就全局铺开。

## 5. 解码器这块的方向

- 和编码器相比，解码器相对“自由一些”，因为它更多承担把高层特征恢复到分割输出空间的任务。
- 课上提到 `FPN`，说明一个很自然的方向是沿着**多尺度、自顶向下融合**来做。
- “上层的加到下层去”这句话可以理解成：把高语义层的信息逐步传到分辨率更高的层，帮助恢复细节。
- 所以如果前面暂时不想大动 backbone，解码器和融合层反而是更适合先试改动的地方。

## 6. 这节课给后续实验的启发

- 第一阶段可以先遵循“**编码器少改，融合模块先做，解码器稳一点**”这个原则。
- 第二阶段再去看更先进的 attention 机制，尤其是最近 `CVPR / ICPR` 里和 RGB-D、多模态、特征交互相关的方法。
- 如果后面要做实验记录，可以优先按下面这个顺序整理：
  - 选什么 backbone
  - depth 是怎么注入的
  - 融合模块放在哪几层
  - 解码器是否沿用 FPN 风格
  - 最后 mIoU 到了多少

## 7. 现阶段最适合落地的一个小结

- 先不要急着把整套模型推翻重来。
- 更现实的路线是：
  - 用成熟 backbone 做基础
  - 优先研究 depth 与 RGB 的 attention 融合
  - 保持接口维度稳定，方便替换模块
  - 解码器先用相对稳妥的结构
- 这样做的好处是：实验变量更少，结果更容易分析，也更适合后面继续往论文方向收敛。
